Hugo + GitHub Pages 搭建个人博客教程(Win)#

一、准备工作#

  1. 注册 GitHub 账号
  2. 安装 Git
    • Windows：git download
  3. 安装 Hugo Extended 版
    • https://github.com/gohugoio/hugo/releases/latest
    • 将下载的 hugo.exe 放置在任意目录，如 C:\hugo\hugo.exe

三、发布到 GitHub Pages（安全方式）#

为什么需要两个仓库？#

在 GitHub Pages 的设置中，有两种方式发布博客，各有优缺点：

  1. 单仓库方式（源码+静态文件混合）：

    • 将整个 Hugo 项目（包括配置文件、主题、markdown 源文件）推送到 GitHub
    • 通过 GitHub Pages 设置从 /docs 目录发布
    • 缺点：任何人都可以查看你的博客源码、配置、markdown 文件内容

  2. 双仓库方式（推荐）：

    • 源码仓库：存放 Hugo 项目的所有源文件（配置、主题、markdown 文章等）
    • 静态文件仓库：只存放生成的静态文件，用于 GitHub Pages 服务
    • 优点：源码与静态文件分离，保护隐私

创建两个 GitHub 仓库#

  1. 源码仓库：blog-source - 存放 Hugo 源码（可设为私有以保护源码）
  2. 静态文件仓库：[USERNAME].github.io - 用于 GitHub Pages 服务

设置 GitHub Pages#

在 [USERNAME].github.io 仓库 Settings → Pages → Source 选择 “Deploy from a branch” 分支选择 main，目录选择 /docs

五、使用 GitHub Actions（推荐）#

如果你希望自动化部署过程，可以设置 GitHub Actions，这样你只需要推送源码到源码仓库， GitHub 会自动构建并部署到 GitHub Pages 仓库：

  1. 在源码仓库（如 blog-source）中创建 .github/workflows/gh-pages.yml 文件：

    name: GitHub Pages

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
        with:
          submodules: true
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v2
        with:
          hugo-version: 'latest'
      - name: Build
        run: hugo
      - name: Deploy
        uses: peaceiris/actions-gh-pages@v3
        with:
          personal_token: ${{ secrets.PERSONAL_TOKEN }}
          publish_dir: ./docs
          publish_branch: main
          external_repository: [USERNAME]/[USERNAME].github.io

首先需要将上述 yml 中的 external_repository 改为自己的 GitHub Pages 仓库，[USERNAME]/[USERNAME].github.io。

其次我们需要从源码仓库推送到GitHub Pages 仓库，需要特定权限，要在 GitHub 账户下 Setting - Developer setting - Personal access tokens 下创建一个 Token。需要开启 repo 与 workflow权限。

使用 GitHub Actions 后，当你推送源码到 blog-source 仓库时，GitHub 会自动：

  • 拉取你的源码
  • 使用 Hugo 生成静态文件
  • 将生成的静态文件自动推送到 [USERNAME].github.io 仓库
  • 更新你的 GitHub Pages 博客

这样你只需要维护源码仓库，所有部署工作都自动完成。
